TCP (Transmission Control Protocol)

TCP is a connection-oriented protocol that ensures reliable communication between devices on a network. It guarantees the delivery of data in the same order it was sent.

Key Features:

  • Reliable Delivery: Ensures data is delivered accurately and in order.
  • Flow Control: Manages the pace of data transmission between sender and receiver.
  • Error Checking: Detects and retransmits lost or corrupted data.

IP (Internet Protocol)

IP is responsible for addressing and routing packets of data so they can travel across networks and arrive at the correct destination.

Key Features:

  • Addressing: Assigns unique IP addresses to devices on a network.
  • Routing: Determines the best path for data to travel across networks.
  • Fragmentation: Breaks down large data packets into smaller ones for transmission.

UDP (User Datagram Protocol)

UDP is a connectionless protocol that allows data to be sent without establishing a connection. It's faster but less reliable than TCP.

Key Features:

  • Low Latency: Faster data transmission without the need for connection setup.
  • Simple: Minimal error checking and no flow control mechanisms.
  • Efficient: Suitable for applications where speed is more critical than reliability.
